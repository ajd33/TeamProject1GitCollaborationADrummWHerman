Branch

“Branch early, branch often” is a good rule of thumb to go by to make your work better organized. Git branch puts your commits into their own branch, for example you could have a branch called “newImage”. By using git branch, you can work on “newImage” making its own commits separately from the master branch then later merge it into the master branch when it is ready. Also, remote repositories of your local one are considered a remote branch which is usually called the origin.

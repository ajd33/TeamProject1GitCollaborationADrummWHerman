Fork

A fork is a copy of a repository, when you fork you can freely make changes without impacting the original. A similar process is copying a file from one drive to another. Forking gives developers access to other developers code, whether to improve it or reuse it in their own project. The ability to fork is an option under tools for a repository, so it does not require a command.

Forking has additional purposes as well. For example, if you need a branch to apply a hot fix, you can fork and apply the fix on a different branch. Also, forking can be used to create a sandbox for a project, an exact replica where you work on applying new features without impacting the original.

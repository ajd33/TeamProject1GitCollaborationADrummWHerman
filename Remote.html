Remote

The 3 commands, add/remove/show deal with the staging area. Any file that is modified and that you want to commit should be added to the staging area using the add command. For example, if we made changes to the index.html file and we want those changes to be saved in a commit then we would git add index.html. To add all files that git detects changes on, we can use the command “git add .”. The opposite of git add is git rm. This command removes any file that you specify from the staging area. Git show shows you one or more objects in Git in a user readable format. It expands the details on these git objects.

Laying Out the Git Flow

Vincent Driessen created GitFlow, the branching model for Git. It has grown in popularity due to it's key benefits for a scaling development team:

  • Parallel Development: is made very easy with GitFlow by isolating working "branches" of a project away from the finished work
  • Collaboration: multiple developers can work on the same feature because each branch is branched off from the project and dependent on only it's own code
  • Release Staging Area: development branches are very key as well to a project's progression. You do not always want to release products to the production system but you want the completed work on standby for the next push.
  • Supports Hot Fixes: sometimes in production systems a problem occurs and you need to patch a leak. You can create a branch that contains a hotfix to a bug then when a permanent solution is ready you can go off the new code.
